Executive Summary

This Terraform module provides comprehensive DNS management on AWS Route53, covering public and private hosted zones, DNS records with advanced routing policies (weighted, latency, failover, geolocation), health checks, DNSSEC zone signing, query logging, and Route53 Resolver endpoints for hybrid DNS resolution. It supports the full spectrum of DNS operations needed for production environments, from simple A records to complex multi-region traffic routing with health-based failover.

Data Flow

  1. Public DNS queries resolve against public hosted zones managed by Route53 global anycast network
  2. Private DNS queries from within associated VPCs resolve against private hosted zones
  3. Routing policies determine which record to return: simple (direct), weighted (traffic split), latency (nearest region), failover (primary/secondary), or geolocation (by location)
  4. Health checks continuously monitor endpoints; unhealthy records are excluded from responses
  5. Alias records resolve directly to AWS resources with zero TTL for instant failover
  6. DNSSEC signs zone responses with KMS-managed keys for DNS response integrity verification
  7. Query logging streams all DNS queries to CloudWatch Logs for analysis
  8. Inbound resolver accepts DNS queries from on-premises networks and resolves them against Route53
  9. Outbound resolver forwards queries matching resolver rules to on-premises or external DNS servers
  10. Resolver rules define per-domain forwarding targets and are associated with specific VPCs

Security Controls

DNSSEC Signing

Zone signing with KMS-managed Key Signing Keys (KSK) ensures DNS response integrity and prevents spoofing.

Private Hosted Zones

VPC-scoped DNS zones prevent internal DNS records from leaking to the public internet.

Health Check Monitoring

Continuous endpoint monitoring with configurable failure thresholds, intervals, and multi-region checks.

Query Logging

All DNS queries logged to CloudWatch for security analysis, compliance auditing, and troubleshooting.

Resolver Security Groups

Resolver endpoints are secured by security groups controlling which networks can send DNS queries.

Failover Routing

Automatic failover to secondary endpoints when primary health checks fail, ensuring availability.

Production Readiness Checklist

  • Enable DNSSEC on public hosted zones for response integrity
  • Configure health checks for all failover and weighted records
  • Enable query logging on critical zones for audit compliance
  • Use alias records for AWS resources (avoids unnecessary DNS hops)
  • Associate private zones with all relevant VPCs
  • Set appropriate TTL values (lower for failover, higher for stable records)
  • Configure calculated health checks for complex availability logic
  • Deploy resolver endpoints in multiple AZs for resilience
  • Set force_destroy = false on production hosted zones
  • Tag all resources for governance and cost tracking
  • Verify NS record delegation at registrar for public zones
  • Test failover scenarios by simulating health check failures

Deployment

module "route53" {
  source = "path/to/terraform-aws-route53"

  name = "my-dns"

  public_zones = {
    "example.com" = {
      comment = "Primary domain"
    }
  }

  private_zones = {
    "internal.example.com" = {
      comment = "Internal services"
      vpc_ids = ["vpc-abc123"]
    }
  }

  records = {
    www = {
      zone_key = "example.com"
      name     = "www.example.com"
      type     = "A"
      records  = ["10.0.1.100"]
    }
    api-primary = {
      zone_key       = "example.com"
      name           = "api.example.com"
      type           = "A"
      records        = ["10.0.1.200"]
      set_identifier = "primary"
      failover       = { type = "PRIMARY" }
      health_check_id = module.route53.health_check_ids["api-check"]
    }
  }

  health_checks = {
    api-check = {
      type              = "HTTPS"
      fqdn              = "api.example.com"
      port              = 443
      resource_path     = "/health"
      failure_threshold = 3
      request_interval  = 30
    }
  }

  tags = { Environment = "production" }
}

terraform init
terraform plan
terraform apply
